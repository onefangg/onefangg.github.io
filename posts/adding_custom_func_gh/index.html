adding silly utils to Github Pages

August 10, 2024

I wanted to try a dumb and quick way to just draft up some silly util functions to speed up some mundane debugging I always do at my work.

Process#

(…or lack thereof?)

My main goal was to get something dirty and hacky; just short of a elegant CICD process.

I have used Hugo-Papermod for the last almost 2 years or-so; so I just needed to get some static JS files boostrapped onto my hugo config and everything else can be left to the utils to do the heavy lifting.

To fast-forward the uninteresting part of the utils:

  • I didn’t want to write the utils in vanilla JS
  • So I wrote the functionailty in Rust -> Compile to WASM -> Browser-facing JS calls the compiled WASM
  • Then build the package

This was my first (!) attempt in trying out Rust -> WASM so I didn’t have time / the know-how to make things perfect; but anyhow the code is out and it’s tracked under this repo.

This is the part where it gets a tad scuffed. The bulk of my hugo configs are only on my PC so there’s not an easy nor sexy way to pull the source code for my scripts and push it nicely to the hugo site generator to handle.

Baby steps - I’ll just zip the built files from the utils and plonk it into the static/ folder within the template. Reading this stackoverflow post helped me figure out the pieces to fit the custom files in.

Following the post, this gets added to my config:

menu:
  main:
    - identifier: fang-utils
      name: fang-utils
      url: "fang-utils/index.html"
      weight: 1

And things fit in (nice!).
